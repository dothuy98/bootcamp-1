tail :
ファイルや標準入力の最後の部分を出力する。 
何も指定しない場合では後方から１０行分出力する。
複数ファイルを指定するとファイル名も出力してくれる。
headというコマンドも存在し、先頭から-nで表示する行数を指定できる。
tailの方がheadより高機能。
（オプション）
-数字 : 標準入力・ファイルの後ろから<数字>の分だけ表示する。
-r : 行単位で逆順に表示する
-c 数字 : <数字>の分だけ文字を最終文字から出力する。改行を１文字に含める。
-r : 行単位で逆順に表示する。
-q : 複数ファイ利を指定してもファイル名を表示させない
-v : 単一ファイルでもファイル名を表示させる。
-f : ファイルの追記を監視するため待機する。
> 変更前と変更後のファイルサイズが同じ場合は反応しない。
　それ以外の削除や追加の場合には変更を出力してくれる
  ctrl + cで終了する。
- : 文字列をずっと打ち続けることができる
-F : -fと似ているが-Fの場合はファイルがリネームされたときリネーム後のファイルを監視してくれる。
     viでの変更はリネーム扱いになりようで-Fでのみ変更を感知してくれる。

(いつ使えそうか)
・ファイルのサイズが大きく見たい情報が後方に集中している場合。
・ログファイルの監視
> tail -F <file_name>
  ログのバックアップファイルを作成した場合、監視対象がバックアップのほうにいってしまうため-Fを使用する。
