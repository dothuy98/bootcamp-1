ps :
現在動作しているプロセスを表示するコマンド。
処理が遅い原因、無駄に実行してしまっているプロセスがないか確認できる。
サーバー関係のプロセスをデーモンと呼ぶ。
なにもオプションを指定しない場合にはその端末で現在動作しているプロセスの時間、PID、コマンドが出力される。

（見方）
UID : ユーザーID
PID : プロセス識別子のこと。 ある時点で存在するプロセスを識別し操作するために使われる。
TTY : 標準入出力となっている端末デバイスを示す
> /pts/0などはターミナルの0番目を示す。
PPID : 親プロセスID。プロセスを開始したプロセス。
VSZ : プロセスの仮想メモリサイズ(単位 : Kizで1Kiz = 1024byte)
RSS : タスクが使用しているスワップされていない物理メモリ（単位 : Kiz）
> スワップ : メインメモリのデータを外部記憶装置（ストレージ）に一時退避させるための領域。
            これによりメインメモリに空きができ、別の処理を行える。
PRI : プロセスの優先順位
NICE : ナイス値。プロセスの優先順位を決定する。
STAT : プロセスのステータスを表示
> S : 割り込み可能な待ち状態。ユーザーの入力待ちやsleepしている場合などでこのステータスになる
  s : セッションリーダー。シェルのプロセスがこのステータスを持ち、プロセスグループを作る。
  　  シェルで実行されたプロセスはこのグループに入り、セッションリーダーのシェルがkillされるとグループに所属するプロセスも同時にkillされる。
  D : 割り込み不可能なスリープ状態。killできない。ディスクやデータバックアップにアクセスする処理で起きることがある。
     この状態で動かなくなったら再起動しかない。
  R : 実行中、実行可能な状態。psで見た瞬間のpsの状態。
  T : プロセス停止状態。ctrl + Zでプロセスを一時停止するとこの状態になる。もしくはkill -STOP PIDでもできる。
  + : フォアグラウンドのプロセスグループに含まれている。バッググラウンドでの処理でないためセッションリーダーのシェル上でコントロール可能。
  < : 優先度が高いプロセス
  N : 優先度が低いプロセス。優先度が高いプロセスの場合は低いものと比べて多くのCPU時間を割り当てられる。
      優先度はナイス値で決定されniceというコマンドでナイス値を変更できる。
      ナイス値は-20から19で定義され、小さいほど優先順位が高い。
      CPU時間が多いと、その分処理が早くなるが、そのほかの優先度が低いプロセスは処理が遅くなる。PRIという項目で管理されている。
  l : マルチスレッドのプロセス。mpiexecなどで並列処理するような大規模な処理を行うプロセス。

（オプション）
-A : 全プロセスを表示
a : 他端末も含めた端末操作のプロセスを表示
x : 実行中のプロセスを表示
c : 実行しているコマンド名を表示
e : 実行しているコマンド名と環境変数を表示する。（すべての環境変数が表示されるため１行がとても長い）
u : CPU、メモリの使用率も表示する。
f : プロセスを階層で表示。プロセスのほとんどがbashから始まることが分かる。
-C : 実行しているプロセス、ファイル名を指定して表示する。

（いつ使えそうか）
・処理が重たいとき。
> ps aux --sort -vsize : メモリの使用量でソートして表示
  ps aux | sort -rk3 : CPUの使用率でソート

・シェルを誤って消したがバックグラウンドで実行しているコマンドのプロセスを削除したい場合。
> 別のシェルから ps -eaf | grep <command>して該当のコマンドのPIDを参照し、kill PIDで削除する
