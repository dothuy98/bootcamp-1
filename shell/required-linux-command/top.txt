top :
プロセスをリアルタイムで監視できる。
topのみを打つとCPU使用順にソートされた状態でプロセスごとの統計情報を確認できる。
NIはナイス値を表す。

（ヘッダーの見方）
1行目 : load average
> 1列目が現在時間、2列目がサーバー稼働時間、3列目がログインユーザの数を表す。
  4列目からが待ちタスクの数を表し、左の数字から1分間あたり、5分間あたり、15分間あたり何個の待ちタスクがあるかを示す。
2行目 : Tasks
> 各状態のタスクの数を見ることができる。
3行目 : CPU
> us : user, sy : system, ni : nice, id : idle, wa : I/Owait, hi : hardware interrupt, si : software interrupt, st : steal
  idle : なにもしていない状態
  I/OwaitはCPUがアイドル状態でシステムに未処理のディスク入出力要求があった時間のこと。
  iowaitの状態ではCPUを使用するプロセスがほかにあればそのプロセスがCPUをつかうことができるためこの％が高いからいってサーバー負荷が高いとは言えない。
  niはniceで実行優先度を変更したプロセスがCPUを消費した時間を表す。
  stはOS仮想化利用時にほかの仮想CPUの計算で待たされた時間を表し、別の仮想CPUが与える影響がどのくらいかを調べることができる。
  OS仮想化は同じOSを使用し、別々の環境を同時に使用する技術のこと。
  似た言葉のハードウェア仮想化はVMWareのようにOS自体が個別で割り振られている。
  表示中に1を押すとCPUごとのデータを見ることができる。
4行目・5行目 : Memory/Swap
> buffers : データを一時的に蓄えておく記憶領域。
  cashed : システムに必要なプログらむーデータを保存する場所。
  ハードディスクなどのストレージ装置は読み込み速度が遅い。
  それを緩和するため使用頻度の高いシステムファイルをキャッシュとして保存することでそのファイルを用いた処理を早く行うことができる。

（プロセス一覧の見方）
VIRT : 仮想メモリ(スワップも含む）
> 仮想メモリ : HDDをメモリ代わりにしてデータ処理を行う場所。
 しかし物理メモリと異なり、データ転送速度がとても遅いためデータを読み出処理の場合は待ち時間が生じる。
REF : スワップしていない使用した物理メモリのサイズ。
SHR : ほかのプロセスと共有されている可能性のあるメモリ。

(操作方法）
qもしくはctrl + cで終了。
h or ?を実行中に押すとヘルプ画面を見ることができる。
u > 文字列入力 > Enterで指定のユーザー名のプロセスのみを表示させる。
k > PID入力 > EnterでプロセスIDをkillできる。topで監視しながらkillできる利点がある
表示中に特定のキーを押すと並びを変更できる。
l : 1行目の表示・非表示を切り替える
t : 2,3行目の表示・非表示を切り替える
m : 4,5行目の表示・非表示を切り替える
(shift + キーで変更する操作方法)
a : プロセス表示を複数画面に切り分ける。
p : CPU使用順にソート。
m : メモリ使用順にソート
r : 昇順・降順を切り替える。
z : カラーとモノクロの切り替え。
w : 設定を保存する
> 設定を破棄する場合はrm ~/.toprcで設定ファイルを削除する。


(オプション)
-a : CPU使用順にソートして表示。
-p <PID> : 特定のプロセスを監視できる。
-u <user> : 特定のユーザーを指定できる。
-d<数字> : <数字>秒ごとに監視する。デフォルトでは3秒で更新される。
-n<数字> : 表示する回数を指定。更新する回数の上限を決める。
-b : バッチモードで実行。するバッチモードはデータを1まとめにして一括処理する形態のこと。
> ファイルにデータをリダイレクトする際に便利。

（使えそうなとき）
サーバーの処理が重たいときなどサーバーが何かおかしいときに実行する。
> top -a
