linuxで作業するにあたり、前提となるような知識について整理する。

ターミナル :
GUIの上でCUIの操作をしたいときに使用するアプリケーションのことを指す。
CUIはコンピューターとのやり取りを文字のみで操作する方法。

また、ターミナル、コンソール、シェルの違いについて
コンソール > 日本語では制御卓という意味で、これはキーボードやディスプレイなどのハードウェアのことを指すが、いまはCUIモードでlinuxを立ち上がたとき黒い画面のことを指す。
ターミナル > GUI上でCUIを操作するときに使うアプリケーションのこと。別名、端末。
シェル > カーネルというOSの中心部分に命令を命令を伝える橋渡しのような役割。linuxのでデフォルトのシェルはbashでほかにはzshなど数字につよいシェルなどがある。

利点として、文字列のみで操作できるのでGUIより素早く、自動化が簡単な点があげられる。


path（パス） :
linuxではディレクトリのツリー構造によってファイルが整理されている。
あるファイルを指定したいとき、それを命令としてコンピュータに伝える方法のことで絶対パスと相対パスという方法がある。

絶対パス >
   必ず/で始まる。最初の/はルートディレクトリを表し、最上位階層を表す。系統樹では根底になる部分でファイルシステムの頂点。
  指定したいファイルがルートディレクトリからどこにあるかを区切りごとに/で表したものが絶対パスの表記方法。

相対パス > 
  カレントディレクトリからファイルが指定したいファイルがどこにあるかを指定する方法。始まりはディレクトリ名（ファイル名）か./で始まる。
  .はカレントディレクトリを表し、カレントディレクトリは自分がディレクトリのツリー構造の内、どこにいるかを指す言葉である。
  簡単にいうと、ディレクト上の自分の現在位置を表す。


ターミナルのショートカットキーについて :
移動 >
  ctrl + b　後退
  ctrl + f　全身
  ctrl + a　行頭へ移動
  ctrl + e　行末へ移動
切り取り・ペースト >
  ctrl + k　行末まで切り取り
  ctrl + u　行頭まで切り取り
  ctrl + y  最後に削除した内容を挿入
  ※クリップボードとは別の場所のレジスタに保存される。したがってctrl + vでペーストはされない。
履歴 >
  ↑(arrow up key)　１つまえのコマンド参照
  ↓(arrow down key)　1つ後のコマンド参照（ない場合は変化なし）
  ctrl + r　コマンド履歴の検索
コマンド実行に対して >
  ctrl + c　実行コマンドの強制終了
  ctrl + z　実行コマンドの一時停止（fgコマンドで復帰できる、一時停止からbgコマンドでバッググラウンドで処理できるように変更もできる）
入力補完 >
  tab　ファイル名を途中まで入力した状態でtabを押すと自動で入力してくれる。
  ファイルが１つしかそのディレクトリにない状態ではtabを押すだけでファイル名を入力してくれる。
