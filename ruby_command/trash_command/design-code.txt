trachコマンド

(概要）
ファイルをrmコマンドで削除すると復元が不可能で間違ったときに取り返しがつかない。
また、いちいちバックアップをとるとコマンドを複数実行することになり面倒なためこのコマンドを作成しようと思った。
イメージはwindowsの削除するとゴミ箱に一時保存することができるような動き。

このコマンドは削除対象に指定したファイルを~/.trashディレクトリに移動する。
その後、指定があれば.zipや.tar.gzに圧縮し、ストレージの圧迫を避ける。
コマンドのmvでも同様のことができるが、やっぱりディレクトリを元の場所に戻したいという場合に手間がかかる。
このコマンドでは元々あった位置も保存するため、1つのコマンドで管理できる。

(デフォルトの処理)
trash <file_name / directory_name>
で~/.trashディレクトリに移動し、移動元をファイルに書き込む。

(オプション）
-r(--restore) <file> : 指定したファイルを元の場所に戻す。圧縮されている場合は解凍する。
-v(--view) : ~/.trashにあるファイル・ディレクトリ名ともともとあった場所を一覧として出力する。
-c(--compress) <圧縮方法> : .zipや.tar.gzなど指定の方法で圧縮してから移動する。
-d(--delete) <file>: 完全な削除を行う。確認をユーザーに行う。
