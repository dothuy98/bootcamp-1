cli2 : logコマンド

目的 : 
作業記録をユーザーのタイミングで残し、時間が空いてしまってもユーザーがスムーズに作業に戻れるようにしたい。
また、作業の記録よりどう自分が作業したかわかるため、無駄な時間の削減を目的にしている。


機能 :
・id、時間、経過時間、メッセージを保存する。

・それらの記録を表示させる。
> 全体、何行出力するか、IDの記録のみを出力、逆順のオプションが考えられる。

・`log.rb . `でカレントディレクトリを簡単に作業の位置として保存する

・.bash_historyのその時点での行数の総数を記録し、そこから次のlogコマンドの実行で増えた分のコマンドを参照する。
> コマンド側ではbash_historyの行数のみ保存しておき、オプションで指定された場合のみ差分の行数に該当するコマンドを出力。
